# GenieClient Governance

GenieClient is an open-source, community-driven MUD client project for DragonRealms on play.net/dr.

GenieClient is independent and is not affiliated with, endorsed by, or operated by Simutronics or play.net.

## Goals

- Keep GenieClient open source and community maintained
- Encourage upstream contributions instead of long-term forks
- Provide a clear path for contributors to become maintainers
- Maintain stability for users, plugin authors, and map contributors

## Maintainer Roles

### Core Maintainers

Core maintainers review and approve changes to primary GenieClient repositories.

### Plugin Maintainers

Plugin maintainers help manage plugin-specific repositories and compatibility.

### Map Maintainers

Map maintainers help review updates to maps, room data, and related files.

### Contributors

Contributors may submit issues, discussions, documentation, pull requests, plugins, maps, and other improvements.

## Decision Making

Most decisions should be made through issues, pull requests, or discussions.

Major changes should be discussed before significant development work begins.

Examples of major changes include:

- Breaking compatibility
- Large architecture changes
- Major plugin API changes
- Release process changes
- Repository restructuring

## Upstream-First Policy

Forks are welcome for experimentation.

Long-term improvements should be proposed back to the official GenieClient repositories whenever possible so the whole community benefits.

## Maintainer Inactivity

If a maintainer becomes inactive, the remaining maintainers may reassign responsibilities to keep the project healthy.

## Community Standard

We assume good intent, encourage respectful collaboration, and prioritize the long-term health of the GenieClient community.
